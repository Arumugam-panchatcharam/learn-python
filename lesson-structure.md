# Lesson Structure

Every lesson in this repository follows the same eight-part format. Use this template when creating or reviewing a lesson.

## Template

### Part 1 — Theory

Explain the concept, when to use it, and how it fits into professional Python development.

### Part 2 — Simple Example

A minimal, runnable example that demonstrates the core idea.

### Part 3 — Professional Example

A realistic example that shows how the concept appears in production-style code.

### Part 4 — Open-Source Example

Inspect code from real libraries to see how experienced developers apply the concept.

### Part 5 — Common Mistakes

Document pitfalls, anti-patterns, and misunderstandings learners often run into.

### Part 6 — Best Practices

Summarize the recommended approach and the reasoning behind it.

### Part 7 — Mini Challenge

A short exercise for the learner to try before moving on.

### Part 8 — Code Review

Walk through a solution (or partial solution) and discuss what good and bad look like.

## Folder Layout

Lessons are grouped by module. Each module has its own folder with lesson subfolders inside:

```
module-01-modern-python/
  README.md
  lesson-01/
    lesson-01.md
  lesson-02/
    lesson-02.md
module-02-functions-advanced/
  README.md
  lesson-11/
    lesson-11.md
```

Add new lessons as `module-NN-<slug>/lesson-NN/lesson-NN.md`.
